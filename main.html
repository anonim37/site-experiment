Build and Test Conflicts in The Wild

Abstract:

Context: : Collaborative software development allows developers to simultaneously contribute to the same project performing different activities. Although this might increase development productivity, it also brings conflicts among developers contributions. Goal: To understand how different contributions from merge scenarios influence build and test conflicts occurrence, in this study we investigate the frequency, causes and adopted resolution patterns. Methodology: We evaluate merge scenarios from Java projects that use Travis CI for continuous integration. To identify conflicts, we access information from git repositories and their associated build process on Travis. Results: We identify more than 150 conflicts. Although most build conflicts happen due to contributor changes, integrator changes performed after integration are also responsible for build conflicts. Unavailable Symbol is the the most recurrent cause of build conflicts. Test conflicts involve not only failed tests but also failed analysis performed after tests execution. Conflicts fixes commonly adapt the project to its new state Conclusion: Our findings show build and test conflicts occur independent of merge conflict occurrence during integration. For build conflicts, we define a catalog of causes that can be applied to assistive tools. For test conflicts, our scripts can be used to support developers during the trial to understand the test case failure.

Replication Study

During this work, we design a study involving three main steps as illustrated bellow. The first step is responsible for getting all information required for the study, which includes mining repositories (source code and builds information). The second step performs many verifications aiming to filter our data of false positive conflicts leaving only true ones. Finally, the last step identifies the causes of build and test conflicts and the resolution patterns adopted (identifying for each conflict the closest commit responsible for the fix).
